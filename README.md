# CS-300-Module-Eight-Journal-Reflection
What was the problem you were solving in the projects for this course?

In this course, the main problem I was solving was how to efficiently store, organize, and retrieve course data for an academic advising system. The goal was to take raw data from a file and build a program that could load that data, sort it in alphanumeric order, and allow a user to search for specific courses. This required not only making the program functional, but also making sure it performed efficiently as the amount of data increased.

How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first understanding the different data structures available, like vectors, hash tables, and binary search trees. I analyzed how each one handled inserting, searching, and sorting data. This helped me decide which structure would be the most efficient for the advising system.

Data structures are important because they directly impact performance. For example, a vector is simple to use but slower when searching large datasets, while a hash table allows faster lookups. A binary search tree can keep data sorted automatically, which is useful for printing courses in order. Understanding these trade-offs helped me make better decisions when designing the program.

How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest roadblocks I ran into was reading the data from the file and making sure it loaded correctly into the program. I also had issues with file paths at first, where the program couldn’t find the CSV file. I fixed this by making sure the file was in the correct directory and testing different input methods.

Another challenge was implementing the sorting and making sure the output was in the correct alphanumeric order. I worked through this by breaking the problem down step by step and testing smaller parts of the code before putting everything together.

How has your work on this project expanded your approach to designing software and developing programs?

This project helped me realize that planning is just as important as coding. Before this course, I would usually jump straight into writing code. Now, I spend more time thinking about the structure of the program and which data structure makes the most sense for the problem.

It also showed me how important efficiency is. Even if a program works, it might not be the best solution if it doesn’t scale well. This changed the way I think about problem-solving and pushed me to consider performance from the start.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project made me more aware of writing clean and organized code. I started using better variable names, breaking code into smaller functions, and adding comments to explain what each part does. This makes the program easier to read and update later.

I also learned to design programs in a way that makes them adaptable. For example, separating the data structure logic from the user interface makes it easier to change or expand the program in the future. Overall, I now focus more on writing code that not only works, but can also be maintained and improved over time.
